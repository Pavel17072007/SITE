# FlashFood

# ⚡ **FlashFood — сервис быстрой доставки еды**

## 📝 Краткое описание  
**FlashFood** — это современная онлайн‑платформа для заказа еды с доставкой на дом или в офис. Сервис объединяет рестораны, кафе и службы доставки, позволяя пользователям быстро находить блюда по вкусу и оформлять заказ в несколько кликов. Удобный интерфейс, продуманный каталог и высокая скорость обработки заказов делают FlashFood идеальным решением для тех, кто ценит время и комфорт.  
FlashFood создан для людей, которые хотят получать любимую еду быстро, удобно и без лишних хлопот.

## 🎯 Целевая аудитория  
- Пользователи, заказывающие еду домой или на работу  
- Люди, ценящие быструю доставку  
- Клиенты, предпочитающие онлайн‑оформление заказов  
- Офисы и компании, оформляющие корпоративные заказы  
- Администраторы ресторанов, управляющие меню  

## 🔑 Ключевой функционал  
- Регистрация и вход пользователей  
- Личный кабинет с редактированием данных  
- Каталог ресторанов и блюд  
- Детальная страница блюда  
- Избранное  
- Поиск по блюдам и ресторанам  
- Корзина и оформление заказа  
- Админ‑панель для управления меню и пользователями  
- Проверка уникальности логина и телефона  
- Динамическая загрузка данных с сервера

# 🛠 Технологический стек

## 🌐 Frontend
### **1. Чистый JavaScript**
**Почему JavaScript**
- Подходит для компактного, динамичного приложения  
- Быстрая загрузка без тяжёлых фреймворков  
- Полный контроль над DOM  

### **2. HTML + CSS**
**Почему HTML и CSS**
- Простая и быстрая разработка  
- Гибкая адаптация под дизайн  
- Лёгкая поддержка  

### **3. Fetch API**
**Почему Fetch API**
- Асинхронные запросы без перезагрузки  
- Современный стандарт  
- Не требует сторонних библиотек  

---

## 🖥 Backend
### **1. Node.js (Express)**
**Почему Node.js**
- Высокая скорость обработки запросов  
- Один язык на фронте и бэке  
- Простая маршрутизация  
- Большая экосистема библиотек  

### **2. MySQL**
**Почему MySQL**
- Идеальна для структурированных данных: пользователи, блюда, заказы  
- Надёжность и стабильность  
- Простые связи между таблицами  

### **3. bcrypt**
**Почему bcrypt**
- Безопасное хранение паролей  
- Защита от brute‑force  
- Удобная интеграция  

### **4. express-session**
**Почему express-session**
- Удобная авторизация  
- Безопасное хранение сессий

# 🍽 Визуальное проектирование интерфейса

## **Главная (/)**  
**Тип доступа:** публичный  

### Основные блоки:
1. Шапка: логотип FlashFood, меню (Главная, Каталог, Войти)  
2. Приветственный блок: заголовок, кнопка «Перейти к ресторанам»  
3. Секция “Популярные блюда”: 6 карточек

# 🗄 **Схема данных и API**

---

## **Модель User (таблица: `users`)**

| Поле | Тип | Описание |
|------|------|-----------|
| id | INT, PK, AUTO_INCREMENT | Уникальный ID |
| login | VARCHAR | Уникальный логин |
| password | VARCHAR | bcrypt‑хеш |
| phone | VARCHAR | Телефон |
| role | VARCHAR | user / admin |

---

## **Модель Product (таблица: `products`)**

| Поле | Тип | Описание |
|------|------|-----------|
| id | INT, PK, AUTO_INCREMENT | ID блюда |
| title | VARCHAR | Название |
| price | DECIMAL | Цена |
| rating | DECIMAL | Рейтинг |
| reviews | INT | Количество отзывов |
| image | VARCHAR | URL изображения |
